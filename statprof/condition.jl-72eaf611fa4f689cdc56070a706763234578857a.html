31 """
32 AlwaysLockedST
33
34 This struct does not implement a real lock, but instead
35 pretends to be always locked on the original thread it was allocated on,
36 and simply ignores all other interactions.
37 It also does not synchronize tasks; for that use a real lock such as [`ReentrantLock`](@ref).
38 This can be used in the place of a real lock to, instead, simply and cheaply assert
39 that the operation is only occurring on a single cooperatively-scheduled thread.
40 It is thus functionally equivalent to allocating a real, recursive, task-unaware lock
41 immediately calling `lock` on it, and then never calling a matching `unlock`,
42 except that calling `lock` from another thread will throw a concurrency violation exception.
43 """
44 struct AlwaysLockedST <: AbstractLock
45 ownertid::Int16
46 AlwaysLockedST() = new(Threads.threadid())
47 end
48 assert_havelock(l::AlwaysLockedST) = assert_havelock(l, l.ownertid)
49 lock(l::AlwaysLockedST) = assert_havelock(l)
50 unlock(l::AlwaysLockedST) = assert_havelock(l)
51 trylock(l::AlwaysLockedST) = l.ownertid == Threads.threadid()
52 islocked(::AlwaysLockedST) = true

81 function _wait2(c::GenericCondition, waiter::Task)
82 ct = current_task()
83 assert_havelock(c)
84 push!(c.waitq, waiter)
85 # since _wait2 is similar to schedule, we should observe the sticky bit now
86 if waiter.sticky && Threads.threadid(waiter) == 0
87 # Issue #41324
88 # t.sticky && tid == 0 is a task that needs to be co-scheduled with
89 # the parent task. If the parent (current_task) is not sticky we must
90 # set it to be sticky.
91 # XXX: Ideally we would be able to unset this
92 ct.sticky = true
93 tid = Threads.threadid()
94 ccall(:jl_set_task_tid, Cint, (Any, Cint), waiter, tid-1)
95 end
96 return
97 end

99 """
100 wait([x])
101
102 Block the current task until some event occurs, depending on the type of the argument:
103
104 * [`Channel`](@ref): Wait for a value to be appended to the channel.
105 * [`Condition`](@ref): Wait for [`notify`](@ref) on a condition and return the `val`
106 parameter passed to `notify`.
107 * `Process`: Wait for a process or process chain to exit. The `exitcode` field of a process
108 can be used to determine success or failure.
109 * [`Task`](@ref): Wait for a `Task` to finish. If the task fails with an exception, a
110 `TaskFailedException` (which wraps the failed task) is thrown.
111 * [`RawFD`](@ref): Wait for changes on a file descriptor (see the `FileWatching` package).
112
113 If no argument is passed, the task blocks for an undefined period. A task can only be
114 restarted by an explicit call to [`schedule`](@ref) or [`yieldto`](@ref).
115
116 Often `wait` is called within a `while` loop to ensure a waited-for condition is met before
117 proceeding.

133 """
134 notify(condition, val=nothing; all=true, error=false)
135
136 Wake up tasks waiting for a condition, passing them `val`. If `all` is `true` (the default),
137 all waiting tasks are woken, otherwise only one is. If `error` is `true`, the passed value
138 is raised as an exception in the woken tasks.
139
140 Return the count of tasks woken up. Return 0 if no tasks are waiting on `condition`.
141 """
142 @constprop :none notify(c::GenericCondition, @nospecialize(arg = nothing); all=true, error=false) = notify(c, arg, all, error)
143 function notify(c::GenericCondition, @nospecialize(arg), all, error)
144 assert_havelock(c)
145 cnt = 0
146 while !isempty(c.waitq)
147 t = popfirst!(c.waitq)
148 schedule(t, arg, error=error)
149 cnt += 1
150 all || break
151 end
152 return cnt
153 end

169 """
170 Condition()
171
172 Create an edge-triggered event source that tasks can wait for. Tasks that call [`wait`](@ref) on a
173 `Condition` are suspended and queued. Tasks are woken up when [`notify`](@ref) is later called on
174 the `Condition`. Edge triggering means that only tasks waiting at the time [`notify`](@ref) is
175 called can be woken up. For level-triggered notifications, you must keep extra state to keep
176 track of whether a notification has happened. The [`Channel`](@ref) and [`Threads.Event`](@ref) types do
177 this, and can be used for level-triggered events.
178
179 This object is NOT thread-safe. See [`Threads.Condition`](@ref) for a thread-safe version.
